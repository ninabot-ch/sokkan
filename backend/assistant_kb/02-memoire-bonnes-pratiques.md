# La mémoire : la discipline qui fait tout marcher
- **Une note = un fichier markdown = UN fait**, avec `description:` en frontmatter. Les sessions les écrivent naturellement ; l'index suit en ~2 minutes (embeddings locaux, rien ne sort de la machine).
- La ligne `description:` est le hameçon du recall — l'écrire comme un sujet de commit.
- Grosses notes = elles pourrissent. Notes atomiques = elles se composent.
- Amorçage d'un nouveau projet : demander à la première session « écris les 5 premières notes mémoire : stack, conventions, lancement des tests, déploiement, pièges connus ».
- Le recall est cross-lingue : notes en FR, questions en EN — ça marche.
- Lier les notes avec [[nom-de-note]] ; les backlinks se calculent tout seuls.
- Si une session semble « amnésique » : vérifier dans Mémoire que les notes attendues existent et que la recherche les remonte (terrain d'essai).
