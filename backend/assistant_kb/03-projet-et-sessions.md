# Importer un projet et travailler
- Importer : créer une session et lui demander « clone <url du dépôt> dans le workspace, puis fais un état des lieux ». La session demande la permission pour chaque commande — c'est le principe.
- Dépôt privé : clé de déploiement en lecture seule (la session guide si on lui demande).
- Rituel quotidien : idée/bug → carte board → ▶ spawn → la session cherche la mémoire, lit le code, propose un PLAN → validation humaine → exécution → la session note ce qu'elle apprend.
- Plusieurs sessions en parallèle : chacune sur son sujet (backend, frontend, infra…) ; la mémoire partagée évite qu'elles se contredisent.
- Permissions : les lectures sont auto-autorisées ; toute mutation (Bash, Edit, Write…) attend un clic. Un plan refusé se reformule — dire pourquoi à la session.
